1 git clone

  • 克隆远程库时指定分支或标签： 

    $ git clone -b <branch>
$ git clone -b <tag>

  • 克隆远程库时指定克隆的深度，这在克隆提交记录很多的存储库时很有用： 

    $ git clone --depth=N

2 git clean

  • 获取要删除的文件列表但不会删除： 

    $ git clean -n

  • 删除当前目录下所有没有 track 过的文件. 但不会删除 .gitignore 文件里面指定的文件夹和文件 

    $ git clean -f

  • 删除指定路径下的没有被 track 过的文件 

    git clean -f <path>

  • 删除当前目录下没有被 track 过的文件和文件夹 

    $ git clean -fd

  • 删除当前目录下所有没有 track 过的文件. 不管他是否是 .gitignore 文件里面指定的文件夹和文件 

    $ git clean -xf

3 git checkout

  • 指定 checkout 时的源分支： 

    git checkout -b <new_branch> <src_branch>

4 git log

  • 指定目标分支： 

    $ git log <branch>

  • 代码行数变动统计，需要在 bash 中执行 

    $ git log --author="$(git config --get user.name)" --pretty=tformat: --numstat | awk '{ add += $1 ; subs += $2 ; loc += $1 - $2 } END { printf "added lines: %s removed lines : %s total lines: %s\n", add, subs, loc }'

  • 指定时间范围 

    $ git log --author="$(git config --get user.name)" --after="2018-01-01" --before="2019-01-01" --pretty= tformat: --numstat | awk '{ add += $1 ; subs += $2 ; loc += $1 - $2 } END { printf "added lines: %s remo ved lines : %s total lines: %s\n",add,subs,loc }'

  • 显示指定提交范围内的历史 

    git log A..B

  • 显示两个 ref 间不同的 commit 

    git log A...B

5 git branch

  • 获取所有远程分支的名称： 

    $ git branch -r

  • 获取所有分支的名称： 

    $ git branch -a

  • 重命名分支： 

    $ git branch -m <oldname> <newname>

  • 获取分支的详细信息： 

    $ git branch -vv

  • 获取包含指定 commit 的所有分支： 

    $ git branch --contains <commit>

6 git push

  • 将所有分支的修改都推送到远程存储库： 

    $ git push --all

  • 将所有标签推送到远程存储库： 

    $ git push --tags

7 git commit

  • 直接提交所有的修改： 

    $ git commit -a

8 git apply

git 提供了两种补丁方案：

  • 通过 git diff 生成的 UNIX 标准补丁 .diff 文件

    这种生成方式将多个 commit 之间的区别生成为一个 diff 文件，不包含提交记录。

  • 通过 git format-patch 生成的 git 专用 .patch 文件

    这种生成方式可以保留 commit 日志，每个 commit 对应一个 patch 文件。

遇到 apply 失败的情况可以尝试如下命令： 

git apply --reject --whitespace=fix mychanges.patch

9 git archive

将 master 以 zip 格式打包到指定文件： 

git archive --format zip --output /path/to/file.zip master

10 git merge

可以使用 –squash 选项将贡献者的 commits 合并为一个 commit 再 merge。这样会让主分支的提交记录看起来更加好看些，同时也可以以 “pull request” 为单位进行责任追踪(git blame)，这对源码的阅读者是非常友好的，但缺点就是丢失了最原始的提交信息，

11 git fetch

拉去远程分支并删除本地分支： 

git fetch -p

12 git stash

暂存（类似栈，最近暂存的在栈顶）： 

git stash <save> <message>

恢复： 

git stash pop
git stash apply

查看： 

git stash list

移除： 

git stash drop

查看 diff： 

git stash show
git stash show -p

参考：

  • git-stash 用法小结 - Tocy - 博客园

13 git cherry-pick

转移一系列 commit： 

$ git cherry-pick A..B

包含 A： 

$ git cherry-pick A^..B
